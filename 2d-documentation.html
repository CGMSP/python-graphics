Animation and Games in Python

 

This is a guide to making interactive animations and games in Python.

 

You need to download the graphics.py library module, which must be in a file named "graphics.py" in the folder that contains your own Python program.

 

You also need to install the Pygame library (if you haven’t done so already).

·      The best way to install Pygame is to use pip from your terminal or command line:

o   pip install --upgrade pygame

·      Pygame is also available from our class Haiku page for download for Windows and older Macs. Newer Macs must use pip.

 

Note: You can press the Esc key to end your program at any time.

 

This page documents version 3.8 of the graphics library.

 

This library was designed and written by Andrew Merrill.

Your First Animation

 

To make an animation or game, you need to write three functions:

  • A function that initializes a new world
  • A function that updates the world for each frame of the animation
  • A function that draws the world on the screen

 

To see how this works, type in and run the following Python program: (or download here: redball.py)

 

from graphics import *

makeGraphicsWindow(800, 600)

 

############################################################

# this function is called once to initialize your new world

 

def startWorld(world):

world.ballX = 50

world.ballY = 300

 

############################################################

# this function is called every frame to update your world

 

def updateWorld(world):

world.ballX = world.ballX + 3

 

############################################################

# this function is called every frame to draw your world

 

def drawWorld(world):

fillCircle(world.ballX, world.ballY, 50, "red")

 

############################################################

 

runGraphics(startWorld, updateWorld, drawWorld)

 

 

 

It should display a moving image of a red circle.

 

How To Specify Colors

 

There are two ways to specify a color:

  1. You can use a string (inside quotation marks) that names the color. Here is a list of available colors that you can refer to.
    For example:     "crimson"
  2. You can specify the red, green, and blue components of the exact color that you want to use. Each component can vary in intensity from 0 (none) to 255 (full intensity). You need to put parentheses around the three component values and separate them with commas.
    For example:     (220,20,60)

 

 

Fundamental Functions

 

You will always need to call following function as the first thing you do in order to make your graphics window:

 

makeGraphicsWindow

Description:

Creates a new graphics window. You must call this function before doing anything else with graphics or animation.

Arguments:

width

the desired width of the animation window, in pixels

height

the desired height of the animation window, in pixels

fullscreen

a boolean - True for full screen mode, False for windowed mode
(optional; defaults to False (windowed))

Returns:

nothing

Examples:

makeGraphicsWindow(800, 600)

makeGraphicsWindow(1024, 768, True)

 

For an interactive, world-based program, your should call this function on the very last line of your program:

 

runGraphics

Description:

Starts the animation or game running. This should be the last thing you do in your program.

Arguments:

startFunction

the name of the function that initializes your new world

updateFunction

the name of the function that updates your world for the next frame

drawFunction

the name of the function that draws the frames of your animation

Returns:

nothing

Example:

runGraphics(startWorld, updateWorld, drawWorld)

Notes:
  • The startWorld function must take one argument (the world).
  • The updateWorld function must take one argument (the world).
  • The drawWorld function must take one argument (the world).

 

For a non-interactive, time-based program, you should call this function on the very last line of your program:

 

animate

Description:

Starts the animation running. This should be the last thing you do in your program.

Arguments:

drawFunction

the name of the function that draws the frames of your animation

timeLimit

the length of your animation in milliseconds

repeat

a boolean - True if your animation should repeat after timeLimit milliseconds, and False if it should end after running once

(optional; defaults to False)

Returns:

nothing

Example:

animate(drawFrame, 6000, True)

Notes:
  • The drawFrame function must take one argument (the number of milliseconds that have elapsed since the animation started).

 

 

You may want your animation to end at some point. The user can always end your program by pressing the Esc key or by closing the graphics window. To have your program make the animation stop, call the following function:

 

endGraphics

Description:

Stops the animation from running.

Arguments:

none

Returns:

nothing

Examples:

endGraphics()

Image Functions

 

loadImage

Description:

Loads a graphics image from a disk file. Unless you want to draw a rectangular image, your image's background should be transparent, or you should specify the color of the image's background for the transparentColor parameter (see below).

Arguments:

filename

the name of the file that contains the image

transparentColor

this optional parameter specifies the background color for the image that should be drawn as transparent on the screen. If the background of your image is already transparent, then do not use this parameter.

Note: If you set this to False, then the image will be treated as opaque, with no transparency. This may improve performance on some computers, especially for full-window backgrounds.
(optional; defaults to None)

rotate

the number of degrees to rotate the image counter-clockwise
(use a negative number to rotate clockwise)
(optional; defaults to 0, meaning no rotation)

scale

the scaling (resizing) factor, where 1 means no scaling, 2 means draw the image at twice its usual size, 0.5 means draw the image at half its usual size, etc.
(optional; defaults to 1, meaning no scaling)

flipHorizontal

a boolean, indicating whether to flip the image horizontally (across the Y axis), where True means flip and False means don't flip

(optional; defaults to False, don't flip)

flipVertical

a boolean, indicating whether to flip the image vertically (across the X axis), where True means flip and False means don't flip

(optional; defaults to False, don't flip)

Returns:

an image that you can pass to the drawImage function (see below) to have drawn on the screen

Examples:

soccerball = loadImage("soccer.jpg")

soccerball = loadImage("soccer.jpg", scale=0.5)

 

drawImage

Description:

Draws a graphic image on the screen

Arguments:

image

the image to draw (must have been already loaded by the loadImage function)

x

the x coordinate of the point on which to center the image

y

the y coordinate of the point on which to center the image

rotate

the number of degrees to rotate the image counter-clockwise
(use a negative number to rotate clockwise)
(optional; defaults to 0, meaning no rotation)

scale

the scaling (resizing) factor, where 1 means no scaling, 2 means draw the image at twice its usual size, 0.5 means draw the image at half its usual size, etc.
(optional; defaults to 1, meaning no scaling)

flipHorizontal

a boolean, indicating whether to flip the image horizontally (across the Y axis), where True means flip and False means don't flip

(optional; defaults to False, don't flip)

flipVertical

a boolean, indicating whether to flip the image vertically (across the X axis), where True means flip and False means don't flip

(optional; defaults to False, don't flip)

Returns:

nothing

Examples:

drawImage(soccerball, 300, 200)

drawImage(soccerball, 300, 200, 90)

drawImage(soccerball, 300, 200, 90, 0.75)

drawImage(soccerball, 300, 200, scale=0.75)

drawImage(soccerball, 300, 200, 45, 0.75, False, True)

Sound Functions

 

We have two categories of audio functions:

 

1.     Sound Functions (loadSound, playSound, stopSound)

a.     intended for short sound clips, like sound effects

b.     supports the WAV and OGG formats, but not MP3

c.     can play multiple sound clips at once

2.     Music Functions (loadMusic, playMusic, stopMusic)

a.     intended for longer music clips

b.     supports MP3 and other formats

c.     can only play one music clip at once

 

loadSound

Description:

Loads a sound clip from a disk file (must be in WAV or OGG format, sorry no MP3s).

Arguments:

filename

the name of the file that contains the sound clip

volume

a number between 0.0 and 1.0 that indicates what volume to use for this sound clip. 1.0 is full volume, and 0.0 would be completely muted.
(optional; defaults to 1.0)

Returns:

a sound that you can pass to the playSound function (see below)

Examples:

beep = loadSound("beep.wav")

soundtrack = loadSound("funkybeat.wav", 0.3)

Notes:

To get free sound clips, record some yourself or try these sites:

  • http://www.ilovewavs.com/Effects/Beeps/Beeps.htm
  • http://www.pacdv.com/sounds/index.html
  • http://simplythebest.net/sounds/WAV/WAV_sounds.html

To record or edit sound clips, try the excellent free program Audacity.

 

playSound

Description:

Plays a sound clip on your computer speakers.

Arguments:

sound

the sound clip to play (must have been already loaded by the loadSound function)

repeat

a boolean indicating whether to repeat the sound forever or just play it once. True means repeat forever, False means play it once.
(optional; defaults to False, play it once)

Returns:

nothing

Examples:

playSound(beep)

playSound(soundtrack, True)

 

stopSound

Description:

Stops a sound clip from playing.

Arguments:

sound

the sound clip to stop

Returns:

nothing

Example:

Mouse Functions

 

There are two ways you can get information from the user's mouse: Event Listeners and Direct Polling. With an Event Listener, you write a function that gets called automatically when the user uses their mouse. With Direct Polling, you can ask about the mouse whenever you want, which you would typically do during each frame in your update world function.

 

You can also control the mouse pointer - showing and hiding it, and moving it around the screen. This could be really annoying, so be careful.

a number indicating which button to check

fireRockets = True

Joystick Functions

 

The joystick functions work with traditional standalone joysticks as well as analog sticks or thumb sticks on gamepads.

 

Each joystick may be able to move in more than one axis. Each axis is treated separately from all of the others.

 

There are two ways you can get information from a joystick: Event Listeners and Direct Polling. With an Event Listener, you write a function that gets called automatically whenever the joystick is moved. With Direct Polling, you can ask what position the joystick is at whenever you want, which you would typically do during each frame in your update world function. If you aren't sure, you probably want to use Direct Polling with a joystick.

 

gameControllerStickAxis

Description:

Returns the current position of a joystick on the given axis. This is the direct polling function for joysticks.

Arguments:

axis

This can be either a number or a string. Either way, it indicates which joystick axis to read.

 

The numbering of the axes will vary from one joystick to another. For example, 0 might be the X axis of the left stick, 1 might be the Y axis of the left stick, 2 might be the X axis of the right stick, and 3 might by the Y axis of the right stick.

 

The names for the axes are determined by the gameControllerSetStickAxesNames() function (see below for details). If you don't set the names yourself, they will default to "X" for axis 0 and "Y" for axis 1, unless your game controller is one that is specially supported by this library. If you have a specially supported game controller, then the system will recognize your controller and automatically determine the names for the axes. You can find out what names the system is using for your axes by called gameControllerGetStickAxesNames() (see below for details).

device

A number indicating which game controller device to use (0, 1, 2, etc.).

(optional; defaults to 0)

Returns:

The current position of the joystick on the indicated axis. The position is a number that will range from -1.0 to 1.0, with 0 meaning that it is centered. Typically, for an X axis, -1.0 will mean far left and 1.0 will mean far right, while for a Y axis -1.0 will mean top and 1.0 will mean bottom. If your joystick twists or has a throttle, then experiment to find out what the values mean.

Example:

stickX = gameControllerStickAxis(0)

stickX = gameControllerStickAxis("X")

speed = gameControllerStickAxis("Throttle")

 

gameControllerNumStickAxes

Returns the number of axes of joystick movement provided by a game controller device. A traditional joystick would provide two axes, one for X motion and one for Y motion. A joystick that can be twisted, or that provides a throttle, might provide three or four axes. A dual action gamepad would provide at least four axes.

Arguments:

device

A number indicating which game controller device to use (0, 1, 2, etc.).

(optional; defaults to 0)

Returns:

the number of joystick axes provided by the given game controller device

numaxes = gameControllerNumStickAxes()

gameControllerSetDeadZone

Description:

Sets a dead zone for the joysticks on your game controllers. Any joystick axis value that is less than the absolute value of the dead zone level will be reported as 0.0. This accounts for the fact that many joysticks will report a resting value that is close to, but not equal to, 0.0. Currently all joysticks will share the same dead zone value, which defaults to 0.05.

Arguments:

deadzone

Returns:

nothing

Example:

Button Functions

 

Your game controller may provide buttons that you can push (a trigger, etc.).

 

There are two ways you can get information from your game controller buttons: Event Listeners and Direct Polling. With an Event Listener, you write a function that gets called automatically when a button is pressed or released. With Direct Polling, you can ask what state a button is in whenever you want, which you would typically do during each frame in your update world function.

Description:

Arguments:

device

(optional; defaults to 0)

Returns:

the number of buttons provided by the given game controller device

numbuttons = gameControllerNumButtons()

Returns the current state of a game controller button. This is the direct polling function for buttons.

device

A number indicating which game controller device to use (0, 1, 2, etc.).

(optional; defaults to 0)

Returns:

a boolean, where True means the button is currently pressed and False means it is not pressed

if gameControllerButton(1) == True:

fireThrusters()
